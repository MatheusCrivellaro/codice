# Architecture Decision Records

Decisões arquiteturais registradas após o MVP. Cada ADR é um arquivo `.md` numerado:
`NNNN-titulo-em-kebab-case.md` (ex: `0001-mover-mensageria-para-websocket.md`).

Crie um novo via `/adr título`. Estrutura padrão:

- **Status** — `PROPOSED` / `ACCEPTED` / `DEPRECATED` / `SUPERSEDED`
- **Contexto** — o problema e as forças em jogo
- **Decisão** — o que foi decidido
- **Consequências** — trade-offs aceitos, o que muda

Decisões fechadas pré-MVP estão em `CLAUDE.md` ("Decisões arquiteturais
fechadas") e em `codice-technical-reference.md §13`. ADRs aqui são para
o que vier depois disso.
