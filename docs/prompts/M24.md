# M24 — Dark mode, microcopy e detalhes finais

Leia o CLAUDE.md antes de começar. Referência: Cap. 6 (paleta modo escuro), Cap. 7 (tom de voz e microcopy) e Cap. 8 (Livro dos Livros — onde aparece) do Documento Mestre em `docs/Codice_Documento_Mestre.pdf`.

## Objetivo

Três frentes de polimento final:
1. Dark mode funcional seguindo a paleta do Doc Mestre
2. Microcopy revisado em TODAS as páginas, alinhado com o vocabulário e tom do Códice

Ao final, o site deve parecer terminado pra um teste com usuários reais — não perfeito, mas coeso e com voz própria.

## 1. Dark Mode

### Paleta (Cap. 6 do Doc Mestre)

Modo escuro:
- Fundo: #1A1714 (tinta de fundo)
- Superfície/cards: #252220 (leve elevação sobre o fundo)
- Texto principal: #E8E0D2 (papel claro)
- Texto secundário: #A89F92 (cinza-quente, mesmo do light)
- Destaque: #B89968 (dourado — substitui o bordô, que "vira sangue" em fundo escuro)
- Dourado hover: #C9AA79
- Bordas: #3D3732

Decisão importante do Doc Mestre: "bordô no light, dourado no dark". Todo lugar que usa bordô no light mode troca pra dourado no dark.

### Implementação

Usar a abordagem do Tailwind + shadcn: `class` strategy no `<html>` (toggle entre `class="dark"` e nenhuma classe).

**Atualizar CSS variables do shadcn** em `index.css` adicionando bloco `.dark`:

```css
.dark {
  --background: 20 12% 9%;        /* #1A1714 */
  --foreground: 33 25% 87%;       /* #E8E0D2 */
  --card: 20 10% 13%;             /* #252220 */
  --card-foreground: 33 25% 87%;
  --primary: 35 35% 57%;          /* #B89968 */
  --primary-foreground: 20 12% 9%;
  --secondary: 33 10% 63%;        /* #A89F92 */
  --secondary-foreground: 33 25% 87%;
  --muted: 20 8% 18%;
  --muted-foreground: 33 10% 55%;
  --accent: 20 10% 16%;
  --accent-foreground: 33 25% 87%;
  --destructive: 0 62% 55%;
  --destructive-foreground: 0 0% 100%;
  --border: 20 8% 22%;            /* #3D3732 */
  --input: 20 8% 22%;
  --ring: 35 35% 57%;             /* dourado */
}
```

Calcular valores HSL exatos a partir dos hex.

**Atualizar tokens customizados do Códice** no @theme ou via media query:

```css
@media (prefers-color-scheme: dark) {
  :root {
    --color-papel: #1A1714;
    --color-papel-profundo: #252220;
    --color-tinta: #E8E0D2;
    --color-tinta-leve: #E8E0D299;
    --color-bordo: #B89968;
    --color-bordo-hover: #C9AA79;
    --color-cinza-quente: #A89F92;
    --color-cinza-borda: #3D3732;
  }
}
```

Mas como vamos usar toggle manual (não só prefers-color-scheme), garantir que os tokens também funcionam com `.dark` class:

```css
.dark {
  --color-papel: #1A1714;
  --color-papel-profundo: #252220;
  --color-tinta: #E8E0D2;
  --color-tinta-leve: #E8E0D299;
  --color-bordo: #B89968;
  --color-bordo-hover: #C9AA79;
  --color-cinza-quente: #A89F92;
  --color-cinza-borda: #3D3732;
}
```

### Toggle de tema

Criar `src/hooks/useTheme.ts`:
- Ler preferência salva em localStorage (key: `codice.theme`, valores: 'light', 'dark', 'system')
- Default: 'system' (respeita prefers-color-scheme do OS)
- Aplicar/remover classe `dark` no `<html>`
- Expor: `theme`, `setTheme(value)`, `resolvedTheme` ('light' | 'dark')

Criar `src/components/ThemeToggle.tsx`:
- Botão com ícone de sol (light) / lua (dark) / monitor (system)
- Dropdown com 3 opções: "Claro", "Escuro", "Sistema"
- Posicionar no header (AppShell), à esquerda do nome do user ou no menu mobile

Adicionar shadcn dropdown-menu se não existir:
```
npx shadcn@latest add dropdown-menu
```

### Revisar componentes que usam cores hardcoded

Percorrer TODOS os arquivos `.tsx` e substituir cores inline por variáveis:
- `bg-[#F7F3EC]` → `bg-papel`
- `text-[#2A2420]` → `text-tinta`
- `text-[#7A2E2E]` → `text-bordo`
- `bg-[#EFE8DA]` → `bg-papel-profundo`
- `text-[#A89F92]` → `text-cinza-quente`

Se alguma cor não estiver no @theme como utility class, adicionar. O objetivo: zero hex inline em componentes. Tudo via token.

### Testar visualmente

Dark mode deve funcionar em TODAS as páginas:
- Home, busca, página do livro, login, cadastro, wizard de anúncio, conversas, painel admin, perfil, 404
- Nenhum texto invisível (texto claro em fundo claro)
- Nenhum componente com fundo branco hardcoded que não troca
- Inputs e textareas legíveis
- Badges de status e condição legíveis
- Fotos/capas continuam visíveis (não aplicar filtro)
- Separadores e bordas visíveis mas sutis

## 3. Detalhes finais

### Favicon placeholder

Até o logo ficar pronto, criar um favicon mínimo:
- SVG simples: letra "c" em Fraunces dentro de um círculo fino, monocromático
- Ou: quadrado com fundo bordô e letra "c" em papel

Criar como `frontend/public/favicon.svg` e referenciar no index.html:
```html
<link rel="icon" type="image/svg+xml" href="/favicon.svg" />
```

### Meta tags finais

Confirmar no index.html:
- `lang="pt-BR"` no `<html>`
- `<meta name="theme-color" content="#F7F3EC" />` (light) com media query pra dark: `<meta name="theme-color" content="#1A1714" media="(prefers-color-scheme: dark)" />`
- og:title, og:description preenchidos

### Scroll suave

Adicionar no CSS global:
```css
html {
  scroll-behavior: smooth;
}
```

### Seleção de texto

Estilizar a seleção pra combinar com a marca:
```css
::selection {
  background-color: #7A2E2E20;
  color: var(--color-tinta);
}

.dark ::selection {
  background-color: #B8996830;
}
```

### Cursor nos links e botões

Garantir que todo elemento interativo tem `cursor-pointer`. Shadcn geralmente cuida, mas verificar links customizados e cards clicáveis.

### Consistência de espaçamento

Revisar que todas as páginas usam o mesmo padrão:
- Padding de página: `py-8` ou `py-12`
- Gap entre seções: `space-y-12` ou `space-y-16`
- Gap entre título e conteúdo: `mb-6` ou `mb-8`

Não precisa ser idêntico em toda página, mas coerente.

## Pegadinhas conhecidas

- Dark mode: shadcn usa `.dark` class strategy. Garantir que o `<html>` recebe a classe, não o `<body>` nem o `<div id="root">`.
- Sonner (toasts) pode não respeitar automaticamente o dark mode. Verificar e ajustar se necessário (Sonner aceita prop `theme`).
- Imagens de capa de livro podem ficar estranhas em dark mode se tiverem fundo branco. Não filtrar — é conteúdo do user. Mas garantir que o container tem border-radius e talvez borda sutil pra delimitar.
- `document.title` via useEffect: lembrar de incluir "— códice" como sufixo em todas as páginas exceto a home.
- Favicon SVG com currentColor se adapta ao dark mode do browser automaticamente se estruturado certo.
- `prefers-color-scheme` e classe `.dark` podem conflitar se o toggle estiver em "sistema" e a classe for aplicada/removida manualmente. O hook useTheme precisa ser cuidadoso: quando em "sistema", ouvir `matchMedia('(prefers-color-scheme: dark)')` e reagir a mudanças.
- Cores inline como `bg-white` ou `text-black` em componentes internos do shadcn vão quebrar dark mode. Revisar os componentes que o shadcn gerou em `src/components/ui/` e substituir por variáveis.
