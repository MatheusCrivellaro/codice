# M22 — Design tokens: fontes, paleta e componentes base

Leia o CLAUDE.md antes de começar. Ler também a seção de Identidade Visual (Cap. 6) e Tom de Voz (Cap. 7) do Documento Mestre em `docs/Codice_Documento_Mestre.pdf` — páginas 16-20.

## Objetivo

Aplicar a identidade visual do Códice no frontend: carregar as fontes da marca (Fraunces, Source Serif 4, Inter), configurar a paleta completa no Tailwind, criar tokens de design reutilizáveis, e estilizar os componentes base do shadcn/ui pra refletir a identidade bibliófila — calma, premium, sem ostentação.

Ao final, o site deve parecer uma livraria curada, não um template de SaaS.

## Contexto

Paleta da marca (Cap. 6 do Doc Mestre):

**Modo claro:**
- Papel: #F7F3EC (fundo principal)
- Papel profundo: #EFE8DA (fundo secundário, cards)
- Tinta: #2A2420 (texto principal)
- Bordô: #7A2E2E (destaque, CTAs, preço)
- Cinza-quente: #A89F92 (texto secundário, bordas)

**Modo escuro (preparar tokens, implementar no M24):**
- Tinta de fundo: #1A1714
- Papel claro: #E8E0D2
- Dourado: #B89968

Tipografia:
- Fraunces (400-500): títulos, display. Google Fonts, serifa moderna.
- Source Serif 4 (400): leitura longa, sinopses, descrições. Google Fonts.
- Inter (400-500): UI, labels, inputs, botões, textos curtos. Google Fonts.

Hierarquia (Cap. 6):
- H1: Fraunces 40-48px
- H2: Fraunces 28-32px
- Corpo/leitura: Source Serif 16-17px, line-height 1.6
- UI: Inter 14-15px
- Pesos baixos (400-500), evitar bold pesado

Layout (Cap. 6):
- Largura máxima de conteúdo: 1280px
- Blocos de leitura: ~680px (70 chars/linha)
- Grid de livros: 4 colunas desktop, 2 mobile, gap 24-32px
- Espaçamento generoso, muito ar
- Cards minimalistas: separação por espaço, não por linha

## O que construir

### 1. Carregar fontes do Google Fonts

Em `frontend/index.html`, adicionar no `<head>`:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Fraunces:ital,opsz,wght@0,9..144,300;0,9..144,400;0,9..144,500;1,9..144,400&family=Inter:wght@400;500&family=Source+Serif+4:ital,wght@0,400;0,500;1,400&display=swap" rel="stylesheet">
```

### 2. Configurar Tailwind

Tailwind v4 usa CSS-based config. Atualizar `frontend/src/index.css` pra incluir os tokens da marca.

Após o `@import "tailwindcss";`, adicionar:

```css
@theme {
  /* Fontes */
  --font-display: 'Fraunces', Georgia, serif;
  --font-body: 'Source Serif 4', Georgia, serif;
  --font-ui: 'Inter', system-ui, sans-serif;

  /* Paleta — Modo Claro */
  --color-papel: #F7F3EC;
  --color-papel-profundo: #EFE8DA;
  --color-tinta: #2A2420;
  --color-tinta-leve: #2A242099;
  --color-bordo: #7A2E2E;
  --color-bordo-hover: #8B3636;
  --color-cinza-quente: #A89F92;
  --color-cinza-borda: #D4CFC7;

  /* Paleta — Modo Escuro (tokens preparados, ativação no M24) */
  --color-fundo-escuro: #1A1714;
  --color-papel-escuro: #E8E0D2;
  --color-dourado: #B89968;

  /* Espaçamento do layout */
  --max-w-content: 1280px;
  --max-w-reading: 680px;
}
```

Isso permite usar classes como `font-display`, `font-body`, `font-ui`, `text-tinta`, `bg-papel`, `text-bordo` etc.

### 3. Estilos globais base

Ainda no `index.css`, após o @theme, adicionar:

```css
body {
  font-family: var(--font-ui);
  color: var(--color-tinta);
  background-color: var(--color-papel);
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

/* Textura sutil de papel (referência Cap. 6: "off-white granulado obtido por textura SVG") */
/* Opcional — só se não atrapalhar performance. Pode ser adicionado depois. */

h1, h2, h3 {
  font-family: var(--font-display);
  font-weight: 400;
  letter-spacing: -0.01em;
}

/* Blocos de leitura longa */
.prose-codice {
  font-family: var(--font-body);
  font-size: 1.0625rem; /* 17px */
  line-height: 1.6;
  max-width: var(--max-w-reading);
}

/* Container de conteúdo */
.container-codice {
  max-width: var(--max-w-content);
  margin-left: auto;
  margin-right: auto;
  padding-left: 1.5rem;
  padding-right: 1.5rem;
}
```

### 4. Customizar tema do shadcn/ui

O shadcn/ui usa CSS variables pra theming. Atualizar as variáveis que o shadcn gerou (provavelmente em `index.css` dentro de `:root`) pra mapear pra paleta do Códice:

```css
:root {
  --background: 40 33% 95%;       /* #F7F3EC convertido pra HSL */
  --foreground: 22 14% 14%;       /* #2A2420 */
  --card: 37 30% 90%;             /* #EFE8DA */
  --card-foreground: 22 14% 14%;  /* #2A2420 */
  --primary: 0 45% 33%;           /* #7A2E2E */
  --primary-foreground: 40 33% 95%; /* #F7F3EC */
  --secondary: 33 10% 63%;        /* #A89F92 */
  --secondary-foreground: 22 14% 14%;
  --muted: 37 20% 85%;
  --muted-foreground: 33 10% 50%;
  --accent: 37 30% 90%;           /* #EFE8DA */
  --accent-foreground: 22 14% 14%;
  --destructive: 0 72% 51%;
  --destructive-foreground: 0 0% 100%;
  --border: 33 15% 82%;           /* #D4CFC7 */
  --input: 33 15% 82%;
  --ring: 0 45% 33%;              /* bordô pro focus ring */
  --radius: 0.375rem;
}
```

Calcular os valores HSL exatos a partir dos hex da paleta. Os valores acima são aproximados — usar ferramenta de conversão pra precisão.

### 5. Estilizar Button

Abrir `frontend/src/components/ui/button.tsx` e ajustar as variantes:

- **default (primary):** fundo bordô #7A2E2E, texto papel #F7F3EC, hover bordô mais claro, font-family Inter, font-weight 500, sem bold pesado
- **outline:** borda cinza-quente, texto tinta, hover fundo papel-profundo
- **ghost:** sem borda, texto tinta, hover fundo papel-profundo
- **link:** texto bordô, underline no hover
- **destructive:** manter padrão (vermelho)

Border-radius sutil (6px), não muito arredondado (não é app mobile). Transição suave no hover (150ms).

### 6. Estilizar Card

Abrir `frontend/src/components/ui/card.tsx`:

- Fundo: papel-profundo #EFE8DA (sutil contraste com fundo #F7F3EC)
- Borda: 1px cinza-borda #D4CFC7 (muito sutil, quase invisível)
- Sem sombra pesada — no máximo shadow-sm com opacidade baixa
- Border-radius: 8px
- CardTitle: font-display (Fraunces), peso 400

### 7. Estilizar Input e Textarea

- Borda: cinza-borda, focus ring bordô
- Font: Inter 14px
- Placeholder: cinza-quente
- Fundo: branco (#FFFFFF) ou papel (#F7F3EC) — branco destaca melhor os campos de form

### 8. Estilizar Badge

Variantes pra contextos do Códice:
- **Status ACTIVE:** fundo verde muito sutil, texto verde escuro
- **Status PENDING:** fundo amarelo sutil, texto amarelo escuro

- **Status SOLD:** fundo bordô sutil, texto bordô
- **Condição (NOVO, COMO_NOVO):** tom verde/emerald
- **Condição (MUITO_BOM, BOM):** tom amarelo/amber

### 9. Estilizar Table (painel admin)

- Header: fundo papel-profundo, font Inter 500
- Rows: hover sutil (fundo com 2% de opacidade a mais)
- Bordas: cinza-borda, finas
- Sem zebra stripes (mantém limpo)

### 10. Estilizar Dialog

- Overlay: preto 40% opacity (leve, não pesado)
- Card do dialog: fundo papel, borda cinza-borda
- Título do dialog: font-display Fraunces

### 11. Estilizar Tabs

- Tab ativa: texto bordô, borda inferior bordô (2px)
- Tab inativa: texto cinza-quente, sem borda
- Font: Inter 14px, peso 500

### 12. Estilizar Toast (Sonner)

Verificar se Sonner respeita as CSS variables do shadcn. Se não, customizar:
- Toast de sucesso: borda verde sutil
- Toast de erro: borda vermelha sutil
- Fundo: papel
- Texto: tinta

### 13. Componente de tipografia reutilizável (opcional mas recomendado)

`src/components/ui/typography.tsx`:

```tsx
// Heading display — Fraunces
export function H1({ children, className }) { ... }  // text-4xl font-display
export function H2({ children, className }) { ... }  // text-2xl font-display
export function H3({ children, className }) { ... }  // text-xl font-display

// Corpo de leitura — Source Serif
export function Prose({ children, className }) { ... } // prose-codice

// Texto UI — Inter (já é o padrão do body)
```

Usar esses componentes nas páginas em vez de `<h1 className="text-4xl font-display">` repetido.

### 14. Aplicar tipografia nas páginas existentes

Percorrer as páginas existentes e substituir:
- Títulos usando `<h1>`, `<h2>` → usar classes font-display ou componentes de tipografia
- Sinopses e textos longos → font-body com prose-codice
- Labels, inputs, botões → já pegam Inter pelo body

Páginas a atualizar (prioridade):
1. `Home.tsx` — hero, seção de curadoria, seção de áreas
2. `BookPage.tsx` — título do livro, sinopse, ofertas
3. `SearchPage.tsx` — resultados, filtros
4. `LoginPage.tsx` e `RegisterPage.tsx` — títulos dos forms
5. `ConversationsPage.tsx` e `ConversationDetailPage.tsx` — headers

NÃO refazer layout neste milestone — só trocar fontes e cores. Layout é M23.

### 15. Preço em bordô

Em todos os lugares que mostram preço (BookCard, ListingOfferResponse, etc), o preço deve ser `text-bordo font-display` — destaque visual sutil, serifa.

## O que NÃO fazer

- Não implementar dark mode (M24 — tokens já preparados, ativação depois)
- Não refazer layout de páginas (M23)
- Não adicionar ilustrações ou mascote
- Não adicionar textura de papel SVG no fundo (nice to have, complexo, pode fazer depois)
- Não trocar ícones (Lucide está ok)
- Não mexer em lógica de negócio, endpoints ou state management
- Não otimizar performance de fontes (font-display: swap já está no link do Google Fonts)
- Não criar design system formal (tokens no Tailwind + componentes shadcn customizados já bastam)

## Critério de pronto

1. Fontes carregam: Fraunces visível nos títulos, Source Serif nas sinopses, Inter nos inputs
2. Paleta aplicada: fundo off-white, texto tinta, botões bordô, cards papel-profundo
3. Home: título "códice" em Fraunces, subtítulo legível, cards de livro com a nova paleta
4. Página do livro: título em Fraunces, sinopse em Source Serif com line-height 1.6, preço em bordô
5. Busca: filtros com visual coerente, resultados com tipografia correta
6. Forms (login, cadastro, anúncio): inputs limpos, botões bordô, focus ring bordô
7. Painel admin: tabela com visual atualizado, badges de status com cores semânticas
8. Nenhum texto usando font-sans genérica onde deveria usar font-display ou font-body
9. Espaçamento generoso entre elementos (não apertado, não esparso)
10. Visual geral transmite "livraria curada", não "template de SaaS" nem "blog WordPress"
11. Zero erros no console
12. Fontes não causam flash visível de texto (FOUT mínimo — o `display=swap` no link do Google Fonts resolve)

## Pegadinhas conhecidas

- Tailwind v4 usa @theme em vez de tailwind.config.js. Se o projeto ainda tiver tailwind.config, os tokens precisam ir no CSS. Verificar como o shadcn configurou e seguir o padrão existente.
- shadcn/ui usa HSL pras CSS variables (formato: `H S% L%` sem a função `hsl()`). Converter os hex da paleta pra esse formato. Ferramenta: https://htmlcolors.com/hex-to-hsl
- Fraunces é uma fonte variável com eixo óptico (opsz). O range 9-144 no link do Google Fonts cobre isso. Se o texto ficar estranho em tamanhos pequenos, pode ser o opsz — testar visualmente.
- Source Serif 4 (não "Source Serif Pro", que é outra). Conferir no link do Google Fonts.
- Preços em Fraunces ficam bonitos mas os números podem ter kerning estranho em algumas combinações. Testar com valores reais (R$ 35,00, R$ 149,90).
- Se as cores ficarem muito sutis (pouco contraste entre fundo e cards), aumentar levemente a diferença entre #F7F3EC e #EFE8DA. O Doc Mestre permite ajuste.
