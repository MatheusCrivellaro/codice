## O que construir

### 1. Variável de ambiente no Vite

O `src/api/client.ts` já usa `import.meta.env.VITE_API_URL` com fallback pra localhost. Confirmar que está assim:

```typescript
const API_BASE_URL = import.meta.env.VITE_API_URL ?? 'http://localhost:8080'
```

Se não estiver, ajustar. Nenhuma outra mudança necessária no client — a env var resolve.

### 2. Criar `.env.production`

Criar `frontend/.env.production`:

```
VITE_API_URL=https://api.codice.com.br
```

Esse arquivo é commitado (não tem segredo — é URL pública). O valor real depende do domínio final. Se ainda não tem domínio, usar a URL do Railway temporariamente:

```
VITE_API_URL=https://<servico>.up.railway.app
```

Criar também `frontend/.env.development` (pra explicitar o dev):

```
VITE_API_URL=http://localhost:8080
```

### 3. Verificar build de produção local

```bash
cd frontend
npm run build
npm run preview
```

`npm run build` gera `dist/`. `npm run preview` serve localmente na porta 4173. Abrir no browser e confirmar que funciona (vai apontar pra API de prod ou Railway — se o backend prod estiver saudável, a home carrega livros).

Verificar:
- Build sem erros nem warnings de TypeScript
- Nenhum `console.log` esquecido em código de produção (não bloqueia, mas limpar o que puder)
- Assets com hash no nome (Vite faz por padrão)

### 4. Configuração da Vercel

Criar `frontend/vercel.json`:

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ],
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    },
    {
      "source": "/(.*)",
      "headers": [
        { "key": "X-Frame-Options", "value": "DENY" },
        { "key": "X-Content-Type-Options", "value": "nosniff" },
        { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" }
      ]
    }
  ]
}
```

Explicação:
- `rewrites`: SPA fallback — qualquer rota serve index.html (React Router cuida do resto). Sem isso, F5 em `/buscar?q=machado` dá 404.
- Cache imutável nos assets com hash (Vite gera nomes tipo `index-abc123.js`)
- Headers de segurança básicos

### 7. Documentação de deploy

Criar `docs/deploy-vercel.md`:

```markdown
# Deploy na Vercel — Frontend

## Setup inicial (uma vez)

1. Criar conta em https://vercel.com
2. "Add New Project" → importar repositório `codice` do GitHub
3. Configurar:
   - Framework Preset: Vite
   - Root Directory: `frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. Environment Variables:
   - `VITE_API_URL`: URL do backend (ex: `https://api.codice.com.br` ou URL do Railway)
5. Deploy

## Verificar

Após deploy, acessar a URL fornecida pela Vercel. Home deve carregar com livros do acervo.

## Deploy contínuo

Todo push na branch main redeploya automaticamente.

## Domínio custom

1. Na Vercel: Settings → Domains → Add Domain
2. Adicionar `codice.com.br` (ou o domínio escolhido)
3. Vercel mostra os registros DNS necessários
4. No Registro.br (ou onde o domínio foi registrado):
   - Tipo A: apontar pra IP da Vercel (76.76.21.21)
   - Tipo CNAME: `www` apontando pra `cname.vercel-dns.com`
5. Aguardar propagação DNS (minutos a horas)
6. Vercel provisiona certificado SSL automaticamente

## Domínio da API (subdomínio)

Para a API responder em `api.codice.com.br`:

1. No Railway: Settings → Networking → Custom Domain → `api.codice.com.br`
2. Railway mostra registro CNAME necessário
3. No Registro.br: CNAME `api` apontando pro valor que Railway indicou
4. Aguardar propagação
5. Atualizar `VITE_API_URL` na Vercel pra `https://api.codice.com.br`
6. Atualizar `CORS_ALLOWED_ORIGINS` no Railway pra incluir `https://codice.com.br`

## Troubleshooting

- **Página em branco:** build falhou. Ver logs na Vercel.
- **404 ao recarregar rota:** vercel.json sem rewrite de SPA. Ver seção 4.
- **API não conecta:** VITE_API_URL errado ou CORS não configurado no backend.
- **Mixed content:** API servindo HTTP e frontend HTTPS. API precisa de HTTPS (Railway já serve).
```

## Pegadinhas conhecidas

- Vercel auto-detecta Vite mas pode errar o Root Directory se o repo for monorepo. Confirmar que está setado pra `frontend`.
- SPA rewrite é obrigatório. Sem ele, qualquer rota que não seja `/` retorna 404 no refresh. O vercel.json resolve isso.
- `VITE_API_URL` é injetada no BUILD TIME (Vite substitui no bundle), não em runtime. Se mudar a env var na Vercel, precisa re-deployar pra aplicar.
- Mixed content: se a API estiver em HTTP e o frontend em HTTPS, o browser bloqueia. Railway serve HTTPS por padrão em domínios *.up.railway.app e custom domains. Mas confere.
- CORS: após configurar domínio custom, atualizar CORS_ALLOWED_ORIGINS no Railway. Esquecer isso é o erro mais comum — o site carrega mas API retorna 403 em CORS preflight.
- Propagação DNS pode demorar até 48h (raro, geralmente minutos). Enquanto não propagar, o domínio custom não funciona — usar URL da Vercel.

Mostre o plano antes de começar.
